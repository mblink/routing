Routing

Once you've built your Routes, they can be used to route HTTP requests to specific application logic. Each route contains enough information to match it against a request and extract the relevant parameters from the request's path and query string, so all you need to do is specify the route-specific handling logic. Assuming you have the test routes:

import routing._

val Login = Method.GET / "login"
// Login: Route[Method.GET.type, Unit] = /login
val Hello = Method.GET / "hello" / pathVar[String]("name")
// Hello: Route[Method.GET.type, String] = /hello/<name: String>
val BlogPost = Method.GET / "post" / pathVar[String]("slug") :? queryParam[Int]("id")
// BlogPost: Route[Method.GET.type, (String, Int)] = /post/<slug: String>/<id: Int>

Then you can specify the handling logic for a given route by calling .handle. The argument passed to with_ should be a function of the type Params => Out where Params is a tuple of the route's parameters and Out is any type you want to target with your handler.

Here's how you could handle the routes above, simply returning a String for each route:

val handledLogin = Login.handle(_ => "Login page")
// handledLogin: Handled.Aux[Login.Method, Login.Params, Login.type, String] = routing.Route$$anon$2@b6a5bbe
val handledHello = Hello.handle(name => s"Hello, $name")
// handledHello: Handled.Aux[Hello.Method, Hello.Params, Hello.type, String] = routing.Route$$anon$2@7915dc36
val handledBlogPost = BlogPost.handle { case (slug, id) =>
  s"Blog post with id: $id, slug: $slug found"
}
// handledBlogPost: Handled.Aux[BlogPost.Method, (String, Int), BlogPost.type, String] = routing.Route$$anon$2@2c8982f6

Of course the actual output type of your handlers will be dictated by the HTTP application framework you've chosen. See the implementations documentation for more examples.
